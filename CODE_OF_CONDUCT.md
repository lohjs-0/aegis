# Code of Conduct

```
> ÆGIS GUARDIAN PROTOCOL
> This community exists to learn, build, and defend — together.
```

## Our Pledge

ÆGIS is a cybersecurity learning platform. We believe security education should be accessible, inclusive, and collaborative.

We pledge to make participation in this project a harassment-free experience for everyone, regardless of age, body size, disability, ethnicity, gender identity, level of experience, nationality, personal appearance, race, religion, or sexual identity and orientation.

---

## Our Standards

**Expected behavior:**

```
→ Be respectful and constructive in all interactions
→ Welcome newcomers — everyone starts somewhere
→ Give and accept feedback professionally
→ Focus on what is best for the community and the platform
→ Acknowledge mistakes and learn from them
```

**Unacceptable behavior:**

```
→ Harassment, trolling, or personal attacks
→ Sexualized language or imagery
→ Publishing others' private information without consent
→ Sharing actual exploits intended to cause real harm
→ Dismissing contributions based on skill level or background
→ Sustained disruption of discussions or pull requests
```

---

## Security Education Context

ÆGIS teaches offensive and defensive security concepts. This means:

- Discussions about real vulnerabilities, exploits, and attack techniques are **welcome and expected** in the context of education
- Sharing payloads, CVEs, and attack analysis for **learning purposes** is encouraged
- Using knowledge gained here to attack real systems without authorization is **strictly prohibited** and against the spirit of this project

```
// The goal is to build defenders, not attackers.
// Understanding attacks is how you build defenses.
```

---

## Enforcement

Instances of unacceptable behavior may be reported by opening an issue or contacting the maintainer directly via GitHub.

All reports will be reviewed promptly and handled with discretion. Maintainers are obligated to maintain confidentiality regarding the reporter of an incident.

Maintainers have the right to remove comments, close issues, or ban contributors who violate this Code of Conduct — temporarily or permanently.

---

## Scope

This Code of Conduct applies to all project spaces — GitHub issues, pull requests, discussions, and any other venue where ÆGIS community interaction takes place.
